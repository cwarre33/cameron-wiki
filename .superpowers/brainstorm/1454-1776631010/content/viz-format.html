Visualization Format — Which output fits best?

All four are achievable with open-source tools. Pick what matches your workflow. Click to select.

A

Interactive Web Dashboard

D3.js force-directed graph, runs locally in browser. Nodes = IPs/orgs/countries, edges = protocol exposure. Filter by protocol, zoom into country clusters. Export to PNG. Best for exploration and demo.

Pros

  • Shareable HTML file
  • Filterable, zoomable
  • Strong portfolio piece

Cons

  • Requires JS to build
  • Heavier upfront work
B

Python NetworkX + Gephi Export

Build graph in Python (NetworkX), export as GraphML or GEXF, open in Gephi for layout. Gephi renders thousands of nodes cleanly. Best for deep graph analysis.

Pros

  • Gephi is purpose-built for this
  • Powerful layout algorithms
  • Good for publication-quality output

Cons

  • Two-step workflow (Python → Gephi)
  • Gephi requires Java
C

CLI → CSV → Plotly/Dash

Pipeline outputs structured CSV/JSON, Plotly Dash serves an interactive choropleth map + bubble chart. Country-level heat map of exposure by protocol. Best for repeatable reporting.

Pros

  • Geographic view is intuitive
  • Easy to re-run on fresh data
  • Dash app is shareable

Cons

  • Less graph topology, more stats
  • Requires Dash server to view
D

Layered: CLI → JSON → Wiki + D3

Pipeline saves structured findings to raw/osint/, auto-ingests into cameron-wiki, AND generates a D3 snapshot. Wiki gets the knowledge; dashboard gets the viz. Best for long-term wiki integration.

Pros

  • Feeds the wiki natively
  • Re-runnable → wiki stays fresh
  • Both research artifact + portfolio

Cons

  • Most moving parts upfront
  • Requires ingest schema design
